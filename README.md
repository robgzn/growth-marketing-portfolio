## Issues Encountered and How I Solved Them

### Issue 1: Unfamiliarity with Code Editors
- **Problem:** I had never used a code editor like Cursor before. The interface looked intimidating at first, with many panels, terminals, and menus I didn't recognize.
- **Solution:** I searched YouTube for a beginner tutorial on Cursor IDE and followed along step by step. It helped me understand the layout and basic functions.

### Issue 2: Finding and Installing Add-ons in Cursor
- **Problem:** I wasn't sure how to find the Extensions panel or search for specific add-ons like Claude Code and Codex.
- **Solution:** I used Cursor's built-in search (Ctrl+Shift+X) to open the Extensions marketplace. Once there, I searched by name and clicked "Install." I also had to log into each service separately, which required creating accounts.

### Issue 3: GitHub Sign-Up and Repository Setup
- **Problem:** I had never used GitHub before and wasn't sure how to create a public repository, what "Public" vs "Private" visibility means, or how to name files properly.
- **Solution:** I searched for "how to create a GitHub repository" and followed a short tutorial. I learned that public repositories are visible to anyone, which is what 100Hires requested. I also learned the naming convention for GitHub files (README.md, lowercase, no spaces).

### Issue 4: Commit and Push Process
- **Problem:** I didn't understand what "staging," "committing," and "pushing" meant — these are Git concepts that were completely new to me.
- **Solution:** I read about the basic Git workflow (stage → commit → push) and used Cursor's Source Control panel which made it visual. The panel clearly showed which files were changed, and I could click the + icon to stage them, write a commit message, and push to the remote repository.

### Issue 5: Markdown Formatting
- **Problem:** The README.md file uses Markdown syntax (headers with #, bullet points with -, bold with **), which I had never used before.
- **Solution:** I looked up a quick Markdown cheat sheet and learned the basic formatting. I tested the preview tab in GitHub to make sure everything rendered correctly before committing.
